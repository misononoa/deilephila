# deilephila 設計ドキュメント

## プロジェクト概要

deilephilaは、テキストベースでP2P分散型のソーシャルネットワークアプリケーションです。
libp2p, IPFSにより投稿を分散配信し、一切のサーバーを必要としません。
未完成であり、現時点ではmDNSを使用したLAN内での配送のみが可能です。
実装のマイルストーン等は [mvp.md](mvp.md) §3 を参照してください。

## 設計コンセプト

- リレー/連合に依存しない真の分散
- 購読ベースのクリーンなタイムライン
- キーペア == アカウントな自己主権のアイデンティティ検証機構
- 投稿・削除・フォローなど全ての操作をイベントとして表現し、各イベントが1つ前のイベントの参照を持つチェーン構造
- IPFS互換

## 技術スタック

- Tauri 2.x
- libp2p
- Vue3 + TypeScript

## 確定した設計判断

| # | 論点 | 決定 |
|---|------|------|
| D1 | 分散アーキテクチャ | libp2p による真のP2P(リレー/連合に依存しない) |
| D2 | アイデンティティ | 公開鍵 = アカウント(Ed25519 鍵ペア、自己主権型) |
| D3 | データ保存 | IPFS(コンテンツアドレス、CIDで参照) |
| D4 | データ構造 | イベントチェーン。投稿・編集・削除・プロフィール・フォローを単一の署名付きイベントとして表現し、各イベントが1つ前のCIDを含む、ユーザーごとの追記専用ログ。現在の状態 = チェーンの fold(イベントソーシング) |
| D5 | MVPスコープ | テキスト投稿・タイムライン + フォロー/フォロワー |
| D6 | head ポインタ(R1) | IPNS に一本化(名前=アカウント公開鍵)。署名付きレコードを gossipsub+DHT で搬送し、取得不能/鮮度不審時はフォローグラフ内の最良レコードを探索。解決は「署名検証OK かつ 最大sequence」の argmax統一規則 |
| D7 | プロフィール可用性 | 表示名を最も複製される IPNS-headレコードに署名付きスナップショット(`display_name` + `profile_cid`)として同梱し、「head発見可⟺表示名描画可」を保証。チェーンの `Profile` が正典でスナップショットは非正規化キャッシュ |
| D8 | 複製・保持(R2) | クライアントが複製範囲/保持期間を任意設定。整合性は署名+CIDで不変(複製ポリシーと無関係)、可用性は創発的・無保証として受容。デフォルト=フォロー相手を上限内で複製＋最新IPNS-headレコードは常時保持(ポインタ/ブロックを分離保持)、自チェーンは常時ピン留め |
